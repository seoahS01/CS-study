### Q. 트랜잭션이란 ?

트랜잭션은 데이터베이스의 상태를 변경하는 여러 작업을 하나의 논리적인 단위로 묶어 처리하는 개념입니다.

데이터베이스에서는 하나의 기능이 여러 SQL로 나뉘어 실행되는 경우가 많은데, 이 과정에서 일부만 성공하고 일부만 실패하면 데이터의 정합성이 깨질 수 있습니다.

트랜잭션은 이러한 문제를 방지하기 위해 모든 작업이 성공해야만 반영되고, 하나라도 실패하면 전체를 이전 상태로 되돌리는 방식으로 동작합니다.

</br>

**💡 왜 중요할까요?**

트랜잭션이 없다면 시스템 장애나 예외 상황에서 다음과 같은 문제가 발생할 수 있습니다.

- 계좌 이체 상황 : 돈은 빠졌는데 상대 계좌에는 입금되지 않는 경우
- 온라인 주문 상황 : 주문은 생성되었지만 재고 차감이 안 된 경우
- 데이터 저장 과정 : 여러 테이블에 걸친 작업 중 일부만 반영된 경우

→ 즉, 트랜잭션은 부분 실패로 인해 **데이터가 불일치 상태로 남는 것을 방지**하는 역할을 합니다.
