<h2>Redis와 MongoDB의 차이점은 무엇인가요?</h2>

>Redis와 MongoDB는 둘 다 NoSQL이지만 카테고리 자체가 다르다.
> 
>Redis는 인메모리 Key-Value 저장소이다. 모든 데이터를 RAM에 올려두고 동작하기 때문에 읽기/쓰기 속도가 매우 빠르다. 
> String, List, Hash, Set, Sorted Set, Stream 같은 다양한 자료구조를 제공한다. 또한 Pub/Sub이나 분산 락 같은 기능도 지원해서 캐시, 세션 저장소, 실시간 랭킹, 메시지 큐 등에 주로 쓰인다.
>
>반면, MongoDB는 디스크 기반 Document 저장소이다. 데이터를 BSON이라는 JSON 유사 형식으로 저장한다. 
> 스키마가 유연해서 필드를 자유롭게 추가할 수 있고, 복잡한 쿼리와 인덱싱, Aggregation Pipeline을 지원하기 때문에 실제 서비스의 주 데이터베이스로 사용된다.
>
>핵심적인 차이는 저장 위치, 데이터 모델, 쿼리 복잡도, 영속성 수준 네 가지이다. 
> Redis는 메모리 중심이라 빠르지만 용량이 제한적이고, MongoDB는 디스크 기반이라 대용량을 저장할 수 있고 복잡한 쿼리도 가능하다. 
> 
>실무에서는 둘을 보완하여 함께 쓰는 경우가 많다. MongoDB에 데이터를 저장하고, Redis는 그 앞단에 캐시로 배치해서 응답 속도를 높이는 구조가 대표적이다.
