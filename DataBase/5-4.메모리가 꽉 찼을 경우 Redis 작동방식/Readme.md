<h2>메모리가 꽉 찼을 경우 Redis는 어떻게 동작되나요?</h2>

>Redis는 `maxmemory` 설정으로 최대 메모리를 제한할 수 있고, 이 한도에 도달하면 `maxmemory-policy` 설정에 따라 동작한다.
>기본값은 `noeviction`으로 새로운 쓰기 요청을 거부하고 에러를 반환한다. 
>하지만 캐시 용도로 사용할 때는 보통 `allkeys-lru`나 `allkeys-lfu`를 사용해서 오래되거나 덜 사용된 키를 자동으로 제거하도록 설정한다.


<h3>`maxmemory`와 `maxmemory-policy`</h3>
Redis는 디스크가 아니라 메모리에 데이터를 저장하기 때문에 메모리가 고갈되면 OS 차원에서 OOM Killer가 Redis 프로세스를 죽여버릴 수도 있다.
이를 막으려고 Redis 자체적으로 메모리 상한선을 걸 수 있는데 그게 `maxmemory`이다.

<h3>Redis의 근사 LRU</h3>

일반적인 LRU는 모든 키를 이중 연결 리스트로 관리해서 접근할 때마다 맨 앞으로 옮기고 제거할 땐 맨 뒤의 키를 뺀다. 
이게 정확하지만 각 키마다 prev/next 포인터를 저장해야 해서 메모리 오버헤드가 크다.

Redis는 근사 LRU를 사용한다.
키들 중에서 N개를 무작위로 샘플링하고, 그 중 가장 오래된 키를 제거한다. 샘플의 수를 늘릴수록 정확도는 높아지고, 성능은 낮아진다.

각 키는 24비트 LRU 클럭 필드만 가지고 있어서 메모리 효율적이다. 
Redis 3.0부터는 '풀(pool)'이라는 개선된 방식을 써서 이전 샘플링에서 찾은 좋은 후보들을 재사용해 정확도를 높였다.
